ACUnSaveAlert — 未保存离开提示

监听表单变化，当用户尝试离开页面时弹出确认提示，防止数据丢失。

1. 监听表单变化

修改下方表单后，尝试点击「返回首页」或刷新页面，会弹出未保存提示。 




<form data-component="unsavealert">
  <input name="title" value="原始值">
  <textarea name="content">内容</textarea>
  <button type="submit">保存</button>
</form>

<!-- 表单被修改后，关闭页面或跳转时自动提示 -->

2. 手动重置已记录的状态

保存成功后调用 resetFormChangedState 让组件"认为表单未变化"

import { resetFormChangedState } from '../src/Auto/ACUnSaveAlert.js';

// 保存 API 成功后，重置状态，防止误提醒
saveApi(data).then(() => {
  resetFormChangedState(formElement);
});

data-* 属性说明

属性说明
data-component="unsavealert"绑定到 form 元素，监听内部字段变化
注意：ACUnSaveAlert 通过监听 beforeunload 事件拦截页面关闭/刷新/跳转。 表单提交或手动调用 resetFormChangedState 后，拦截会自动解除。 仅在用户实际修改了表单字段值时才会触发提示。
